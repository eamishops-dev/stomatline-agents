# Logistics — 2026-07-06
### LOG-1. Зворотний рейс не порожній
Користувач: водій/логіст; опосередковано склад і закупівельник.
Біль (час/енергія/гроші): рейс повертається порожнім; повернення, прострочені партії, підписані документи чекають «наступного разу» → окремі поїздки (cost-to-serve ↑, гроші) або зависання тижнями; клініка тримає непотрібну коробку (довіра після доставки ↓).
Рішення: у маршрутний лист кожної зупинки Odoo додає задачі зворотного рейсу: забрати повернення / прострочку / документи; водій відмічає одною кнопкою «забрав / не було»; склад заздалегідь бачить, що приїде.
Доставка в робочий процес: Odoo batch picking / route manifest + return picking, згенерований до виїзду.
Режим виконання: Гібрид (система формує список, водій виконує).
Метрика болю: % рейсів зі зворотним завантаженням; окремих поїздок за поверненнями/міс → 0; cost-to-serve по маршрутах.
